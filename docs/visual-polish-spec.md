# Vibe Coding 课程视觉改进 Spec（完整审核版）

更新时间：2026-07-16  
状态：已执行，浏览器验收通过（`file://` 受浏览器安全策略阻止，已完成静态路径核对）  
主交付文件：`AI-Coding-Course/interactive-lecture.html`

## 1. 项目目标

把现有课件从“深色科技网页”提升为“科技杂志 × 电影化演讲”体验。改进重点不是增加装饰，而是建立明确的视觉层级、页面节奏和情绪起伏，使非技术观众在现场投屏时能快速抓住每一页的核心信息。

最终体验应满足：

- 开场像案例预告片，而不是视频素材列表。
- 中段像一套有章节节奏的演讲，而不是连续的后台卡片页面。
- Token、Prompt、Context、Harness 等概念各有独立构图，但仍属于同一视觉系统。
- 尾声进入独立的电影化舞台，不再保留网页导航和普通卡片感。
- 1440×900 为主要投屏验收尺寸，兼顾 1920×1080 和 390×844。

## 2. 已确认的边界

- 保留深色背景、超大标题、Token 动态图形、青色与金色对比。
- 普通章节左侧继续保留竖排 `TOKEN` 装饰。
- 只移除左侧装饰中的“想象力、好奇心、学习力、执行力”。课程正文和“四力”章节中的这些内容继续保留。
- 不改写课程核心观点，不删除现有章节、案例、互动、视频、音频、人物或尾声图片。
- 允许为了视觉节奏重新分组、渐进展示或改变同一章节内部的呈现顺序，但不得丢失内容。
- 不新增后端、外部 API、第三方 UI 框架或在线字体依赖。
- 主交付仍为可离线运行的单文件 HTML；新增视觉资源必须保存在仓库内。

## 3. 视觉方向

### 3.1 关键词

- 科技编辑感
- 克制、清晰、有观点
- 大尺度排版
- 非对称构图
- 电影化尾声
- 少卡片、强层级

### 3.2 应保留的现有特征

- 接近黑色的背景场域。
- 米白色大标题与高对比正文。
- 青色代表技术、行动和核心概念。
- 金色代表人物、历史、价值与尾声。
- 红色只代表错误、风险和警告。
- Token 圆环、章节大号数字和尾声素材继续作为记忆点。

### 3.3 应消除的现有问题

- 顶部导航入口过多，投屏时像文档网站。
- 中段大量页面采用同一种“标题 + 灰色卡片网格”构图。
- 多种强调色在标签和卡片中平均出现，缺少主次。
- 开场原生播放器控件和等宽卡片抢走标题注意力。
- 尾声仍能看到网页导航、全局进度和滚动条。
- 人物仍被装进普通卡片，没有形成电影化人物场景。
- 尾声文案、人物姓名、肖像之间的尺度差不够明显。

## 4. 详细需求

### R1. 顶部导航改为演讲控制条

#### 设计要求

普通浏览状态下，顶部只保留以下五类信息：

1. 品牌标识。
2. 当前章节名称。
3. 当前章节序号，例如 `05 / 14`。
4. “课程地图”入口。
5. 全屏与回顶操作。

不再长期平铺所有章节名称。点击“课程地图”后，以抽屉或覆盖层展示完整主章节和附录入口；关闭地图后恢复当前章节控制条。

#### 具体落点

- 主导航不直接展示 Harness 附录、工具与模型附录、效率附录等低频入口。
- 当前章节名称随滚动更新，不要求用户从横向导航中寻找当前位置。
- 课程地图必须包含所有可访问章节，不能因为主导航精简而失去入口。
- 桌面端控制条保持一行；移动端可以隐藏品牌文字，但保留当前章节、地图和全屏。

#### 验收标准

- 1440×900 下导航无截断、无横向滚动、无被遮挡的文字。
- 任意主章节和附录均可从课程地图到达。
- 当前章节名称与页面实际位置一致。
- 键盘焦点可进入地图、关闭地图并返回触发按钮。

### R2. 建立四种页面构图，打破卡片堆叠

#### 设计要求

正文不再全部使用同一种卡片网格，而是建立四种明确页面原型：

1. **章节幕**：一个超大标题、一句核心判断、极少辅助信息。
2. **概念页**：一个主视觉或主图形配一组解释，不出现平均分布的卡片墙。
3. **案例页**：真实素材占主导，文字承担解释而不是与素材等权竞争。
4. **互动页**：问题、操作、反馈分步出现，同一时刻只强调当前动作。

#### 具体落点

- Why、Token、Prompt、Context、Harness、总结页应采用不同的主构图，不得只更换标题和标签颜色。
- Harness 当前的大量信息必须分层：先显示“四个工程条件”，再渐进进入自动化系统和 Skill 示例；不能首屏同时出现所有卡片。
- 总结页保留“一个好 Prompt 的六个零件”，但六项应形成一张完整参考图，而不是六张彼此独立的普通卡片。
- 超过一屏的章节允许保留纵向内容，但首屏必须完整显示该章节的标题、核心结论和主要视觉或主要操作。
- 附录可以更密集，但必须通过表格、对照、流程或分组排版呈现，避免继续复制主课卡片网格。

#### 验收标准

- 主课相邻两章不能连续使用相同的标题位置、列数和卡片布局。
- 1440×900 下，每个主章节首屏都能看到完整核心结论。
- Harness 首屏不同时出现全部工程细节。
- 总结页在一张截图中可以被完整保存和阅读。
- 页面中不存在无层级的三列或四列“平均卡片墙”作为主要构图。

### R3. 左侧 Token 装饰

#### 设计要求

- 普通主章节继续保留竖排 `TOKEN`。
- 删除装饰层中的“想象力、好奇心、学习力、执行力”。
- 不删除正文或“四力”章节中的四种能力内容。
- `TOKEN` 只承担品牌化章节标识，不再与正文争夺注意力。

#### 具体落点

- 桌面端 `TOKEN` 使用低亮度青色，位置固定在左侧安全区。
- 章节内容与 `TOKEN` 保持足够间距，不出现标题压住装饰的情况。
- 560px 以下隐藏该装饰，避免占用移动端正文空间。

#### 验收标准

- 普通桌面章节可见 `TOKEN`。
- 装饰伪元素不再输出四种能力文字。
- “四力”章节正文仍完整存在。
- 移动端无侧栏引起的横向溢出。

### R4. 开场案例改为主案例预告页

#### 设计要求

开场采用“一个主案例 + 两个辅助案例”的非对称构图，不再是三个等宽播放器。普通妈妈案例作为主案例，错题整理和运动识别作为辅助案例。

#### 具体落点

- 三个视频使用统一视觉比例和统一标题基线。
- 主案例获得更大的画面面积和更强标题，辅助案例缩小但仍可独立播放。
- 未播放时优先显示统一封面与播放按钮，弱化浏览器原生控制条；开始播放后再显示必要控制。
- 竖屏或比例不同的视频使用裁切、模糊背景或留白框解决，不允许出现随机黑边和尺寸跳动。
- 封面如需新增，应从现有视频提取本地帧，不使用远程图片。
- “看完案例，开始交流”是本页唯一主行动按钮。

#### 验收标准

- 1440×900 下同时看见标题、三个案例入口和开始交流按钮。
- 主案例与辅助案例的面积关系明显，不能仍然看起来等权。
- 未播放状态下原生控制条不成为视觉主角。
- 三个视频均能播放、暂停、拖动，并保留键盘可操作性。
- 视频加载前后卡片高度不跳动。

### R5. 尾声进入完整电影模式

#### 设计要求

尾声不是普通章节，而是独立的全屏电影舞台。进入尾声可视区域后自动切换视觉模式；背景音乐仍遵守浏览器策略，在用户第一次按右方向键后开始播放。

#### R5.1 自动隐藏网页界面

- 尾声进入可视区域后自动隐藏顶部导航。
- 自动隐藏全局进度条。
- 自动隐藏浏览器页面滚动条，锁定纵向滚动。
- 尾声舞台占满 `100vw × 100vh`，不得露出上一章节。
- Esc 退出尾声模式后恢复导航、全局进度条、滚动条和原滚动位置。

#### R5.2 人物肖像改为大面积渐隐背景或无边框构图

- 莎士比亚不再放在右上角普通卡片中；头像改为右侧大面积半身或圆形渐隐背景，与标题叠合构图。
- 单人物画面中，肖像应占画面宽度约 32%–45%，使用灰度、暖色或低饱和处理，并通过渐变蒙版融入背景。
- 多人物画面不再使用四张独立人物卡片；改为无边框人物阵列、重叠肖像或编辑式拼贴。
- 肖像边缘不能出现普通 UI 卡片边框、圆角面板或投影盒子。
- 姓名和身份作为轻量文字标注，不能重新变成信息卡片。
- 所有人物图片继续使用本地资源并保留可访问的替代文字。

#### R5.3 强化文案与人物的尺度反差

- 短句尾声文案回到 Spec 前的紧凑节奏，使用 `clamp(28px, 3.2vw, 42px)`，不再用超大字号挤压肖像同屏空间。
- 长段文案使用 `clamp(20px, 2.2vw, 31px)`，最大宽度约 820px，并与人物顶部对齐，避免字幕整体下坠。
- 多人物肖像采用两列无边框紧凑阵列；单个头像约 `clamp(88px, 7vw, 112px)`，姓名与身份说明在头像旁同屏显示。
- 人物姓名约 15px，身份说明约 11px；主要文案仍明显大于人物标注，但不牺牲姓名可见性。
- 文案占据主视觉层，肖像负责氛围和历史感，不与文案等权竞争。
- 多人物头像统一使用径向渐变遮罩、低饱和滤镜和背景色融边，不能保留硬裁剪边缘。

#### R5.4 只保留极弱的方向键提示

- 删除“按 → 开始播放、按 → 跳过、按 ← 回看、Esc 重新开始”等成排控制说明。
- 初始状态只在底部中央显示一个低亮度 `→` 和极短提示，例如“继续”。
- 提示字号 11–12px、透明度不高于 0.35，不使用高亮按钮、边框或底色。
- 第一次操作后提示自动淡出；仅在用户停留较久时允许极弱地重新出现。
- BGM 状态可保留在右下角，但字号和亮度必须低于正文，不得成为第二组控制栏。

#### R5.5 尾声内部进度

- 全局页面进度条隐藏后，可保留一条不超过 2px 的尾声内部进度线。
- 内部进度线使用低亮度金色或青色，不显示百分比和章节数字。
- 不出现分页圆点墙、时间轴卡片或其他 UI 化控件。

#### 验收标准

- 滚动或跳转到尾声后，无需点击即可隐藏导航、全局进度和滚动条。
- 第一次按右方向键后 BGM 和首幕正常开始。
- 任一尾声截图中不出现人物卡片边框。
- 莎士比亚肖像为大面积渐隐背景或无边框主构图。
- 多人物画面采用无边框阵列或拼贴，不再是四张卡片。
- 短句文案明显大于人物姓名，尺度差可直接从截图辨认。
- 屏幕上只保留极弱方向键提示和必要的 BGM 状态。
- Esc 后导航、滚动条、页面滚动和尾声初始状态全部恢复。

### R6. 字体系统建立明确角色

#### 设计要求

不依赖网络字体，使用 macOS 常见本地字体建立三种角色：

- 技术章节标题：`SF Pro Display`、`PingFang SC`、系统黑体，使用 760–850 字重。
- 正文和互动说明：`PingFang SC`、`Noto Sans SC`、系统无衬线，使用 400–600 字重。
- 尾声、历史人物和引语：`Songti SC`、`STSong`、系统衬线，作为电影化情绪字体；英文仍使用系统 Display 字体。

#### 具体落点

- 技术章节不混用衬线字体。
- 尾声只在标题、引语或关键短句中使用衬线，不把全部长正文改成宋体。
- 章节数字、英文术语和中文标题之间保持清晰的字号等级。
- 字号、行高和字重通过 CSS 变量统一，不继续在大量单独选择器中随意定义。

#### 验收标准

- 技术正课和尾声在字体气质上明显不同，但仍属于同一页面。
- 离线打开时不产生字体请求和布局闪动。
- 1440×900 和 390×844 下无标题溢出或不合理断行。

### R7. 颜色、表面和卡片层级收敛

#### 设计要求

- 青色：核心概念、当前操作、技术连接。
- 金色：人物、历史、价值判断和尾声。
- 红色：错误、风险、禁止和警告。
- 蓝、绿、紫仅在确有语义区分的图表或互动状态中使用，不能作为随机标签装饰。

#### 具体落点

- 每页只允许一个主强调色；次要内容回归灰阶。
- 主视觉面板可无边框或仅使用极弱描边。
- 普通信息组不同时拥有边框、底色、圆角和重阴影四种装饰。
- 卡片圆角、边框亮度、阴影强度形成统一变量。
- 进度条不使用平均彩虹渐变；普通章节以青色为主，尾声以金色为主。

#### 验收标准

- 任意主章节截图中不存在四种以上等权强调色。
- 警告内容仍能通过红色被快速识别。
- 普通卡片不再全部浮在相同灰色面板上。
- 尾声无普通后台面板感。

### R8. 动效只服务于叙事

#### 设计要求

- 每个章节只保留一次有方向的进入动效，用于建立阅读顺序。
- Token 圆环旋转、互动反馈和尾声字幕切换继续保留。
- 删除无意义的随机悬浮、普遍缩放和每张卡片各自动画。
- 动效时长以 450–650ms 为主；不拖慢讲解节奏。

#### 具体落点

- 章节幕按照“章节号 → 标题 → 核心结论 → 主视觉”顺序出现。
- 互动页只对新出现的反馈内容做动效。
- 尾声继续使用上移、缩放、模糊到清晰和遮罩展开，但人物背景变化应比字幕更慢、更弱。
- `prefers-reduced-motion` 下停止旋转和位移动效，内容直接可见。

#### 验收标准

- 同一屏不会同时出现多个无关循环动画。
- 动效关闭后全部内容仍可访问和操作。
- 快速翻页时不出现长期残留的半透明或模糊元素。

### R9. 投屏、移动端和可访问性

#### 设计要求

- 主要验收尺寸：1440×900。
- 辅助桌面尺寸：1920×1080。
- 移动端尺寸：390×844。
- 键盘操作、焦点样式、媒体控制和降低动效偏好继续有效。

#### 验收标准

- 三个尺寸均无横向页面溢出。
- 1440×900 下主章节首屏不截断核心标题或主要按钮。
- 移动端导航可打开课程地图，视频和互动控件可操作。
- 文字与背景保持足够对比，弱提示除外；弱提示不承载必要信息。
- `file://` 和本地 HTTP 两种方式下本地媒体路径均保持可用。

## 5. 内容与资源保护

以下内容不得因视觉重构被删除或替换：

- 三个开场视频及其现有文案。
- Token 分配互动。
- Prompt 诊断互动。
- Context、Harness、上手路径、纳瓦尔、案例和总结内容。
- 尾声全部字幕、人物、BGM、AI Coding 场景轮播和键盘控制。
- 本地人物头像与媒体路径。

允许新增的资源仅包括：

- 从现有视频提取的本地封面帧。
- 为现有图片生成的裁切版本或透明渐变版本。
- 不依赖外部服务的本地纹理或遮罩。

## 6. 实施阶段（已执行）

### 阶段 A：导航与全局视觉系统

- 建立演讲控制条和课程地图。
- 建立字体、颜色、圆角、边框、阴影和动效变量。
- 保留 `TOKEN`，移除装饰四力文字。

### 阶段 B：开场与主课节奏

- 重做开场主次案例构图。
- 为主课章节分配四种页面原型。
- 重构 Harness、总结及其他卡片密集页面的渐进层级。

### 阶段 C：尾声电影模式

- 实现进入尾声即隐藏导航、全局进度和滚动条。
- 重构莎士比亚和多人物肖像为无边框渐隐构图。
- 调整字幕尺度和人物标注尺度。
- 精简方向键提示与 BGM 状态。
- 验证 Esc 完整恢复。

### 阶段 D：完整验收

- 静态结构与脚本解析。
- 桌面、移动端、`file://` 与 HTTP 浏览器检查。
- 视频、音频、人物和尾声轮播检查。
- 逐条核对本 Spec 的 R1–R9 验收标准。
