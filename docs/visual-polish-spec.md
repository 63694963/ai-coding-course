# Vibe Coding 课程视觉改进 Spec

更新时间：2026-07-16  
适用文件：`AI-Coding-Course/interactive-lecture.html`  
目标：把现有深色科技网页提升为适合现场投屏的“科技杂志 × 电影化演讲”课件。

## 已确认的设计边界

- 保留当前的深色背景、超大标题、Token 动态图形、青色与金色对比。
- 第三点的左侧装饰必须保留 `TOKEN`。
- 第三点的左侧装饰移除“想象力、好奇心、学习力、执行力”四行文字。
- 不新增后端、构建器、外部 API 或 npm 依赖。
- 主交付仍是单文件 HTML，媒体路径和本地头像不改名、不外链化。
- 以 1440×900 桌面投屏为主要验收尺寸，同时保持移动端可读。

## 改进范围

### 1. 导航降噪

顶部导航保留品牌、主要章节入口、全屏按钮和回顶按钮，但缩短各入口文案并隐藏低频附录入口。导航必须在 1440px 宽度下不出现右侧截断；附录仍可通过章节地图或正文链接访问。

### 2. 侧栏与信息层级

保留每个普通章节的 `TOKEN` 竖排装饰，删除同一伪元素中的四力文字。章节数字作为主要层级锚点；页面正文不再被四力文字占用左侧视觉空间。

### 3. 开场案例

开场视频改为“一个主案例 + 两个辅助案例”的层级。三个视频仍然可播放，但卡片高度、封面比例、标题和按钮必须统一；1440×900 下标题、视频卡片和“开始交流”入口都应能看到，不要求用户先滚动才能开始课程。

### 4. 尾声与演示模式

进入尾声并开始播放时自动开启演示模式，隐藏顶部导航和全局进度条。尾声保留文案和人物素材，但弱化普通卡片边框，增加舞台留白，保证 1440×900 下呈现为独立的电影化场景。

### 5. 视觉系统收敛

保留既有 CSS 变量体系。青色用于核心概念与交互，金色用于人物和尾声，红色用于风险提示；不新增彩虹式强调色。字体继续使用本地系统字体栈，避免离线投屏时出现外部字体加载问题。

## 执行顺序

1. 更新导航文案和低频入口的可见性。
2. 删除四力侧栏文字，仅保留 `TOKEN`。
3. 调整开场视频栅格、卡片尺寸和开始入口布局。
4. 调整尾声自动演示模式和电影化视觉。
5. 在浏览器中按 1440×900 验收开场、Token、Harness、总结和尾声，并做移动端快速检查。

## 验收标准

- [ ] `AI-Coding-Course/interactive-lecture.html` 中普通章节仍显示 `TOKEN`，不再显示“想象力”“好奇心”“学习力”“执行力”。
- [ ] 1440×900 下顶部导航没有文字被截断，低频附录入口不挤占主导航。
- [ ] 开场案例的三个视频比例一致，开始交流入口在首屏可见。
- [ ] 首次按右方向键进入尾声时，顶部导航和全局进度条隐藏；Esc 重置后可以恢复普通页面状态。
- [ ] 尾声字幕、BGM、人物头像和 AI 场景轮播仍可正常工作。
- [ ] 通过 Node 内联脚本解析、媒体路径检查和 `git diff --check`。
- [ ] 移动端不出现横向滚动，导航和开场视频仍可操作。

## 不在本次范围

- 不重写课程文案和章节结构。
- 不删除视频、音频、人物头像或尾声场景图片。
- 不改动 `AI-Coding-Course/interaction-demo.html`。
- 不新增字体下载、第三方组件或远程资源。
