# 实现计划：AI Coding 课程展示网页

## 需求复述

基于 `/Users/jerry/Documents/repo/讲课/AI-Coding-课程大纲.md`（15 章节，1243 行），制作一个 HTML 网页用于投屏展示。受众为零编程基础的初学者。

### 已验证的设计决策

| 决策点 | 结论 |
|--------|------|
| 页面结构 | C：长滚动 + 顶部导航，可快速跳到任意章节 |
| 视觉风格 | 苹果发布会风格 |
| 内容范围 | A：15 个章节完整呈现 |

### 待定决策

- 视觉方向细化 — 用户表示后续用 Skill 决定，暂不推进

## 苹果发布会风格要素

- 深色背景为主，渐变点缀
- 超大字号 + 宽行距，强视觉层次
- 大量留白，每屏聚焦一个概念
- 平滑的滚动过渡动画
- 毛玻璃/半透明效果用于导航条
- 系统字体栈（SF-style），数字要突出
- 卡片、代码块使用微妙的圆角和阴影
- 色彩克制：主色 + 少量强调色点缀

## 实现阶段

### Phase 1：页面骨架和导航

- 创建单文件 HTML，包含所有 15 个章节的 DOM 结构
- 顶部 sticky 导航栏，列出章节名（hover 高亮、scroll-spy 激活态）
- 平滑滚动（`scroll-behavior: smooth`）
- 章节锚点 id

**验证**：导航栏点击可跳转，滚动时当前章节高亮

### Phase 2：苹果风格 CSS 设计系统

- CSS 自定义属性：颜色、字号、间距、圆角、过渡时间
- 深色背景基调（近黑，不是纯黑），渐变光晕点缀
- 超大标题（`clamp(3rem, 8vw, 8rem)` 量级）
- 正文舒适阅读字号
- 导航栏毛玻璃效果（`backdrop-filter`）
- 卡片、表格、代码块的统一风格
- 章节间的大留白过渡

**验证**：投屏尺寸下（1280×720 或 1920×1080）可读，文字不过密

### Phase 3：内容格式化

- 15 个章节依次排版
- 对比表格（好 prompt vs 坏 prompt）用卡片式设计
- 代码块（prompt 示例）用等宽字体 + 暗色背景
- 引用/重点文字用左侧色条标注
- 章节标题用数字编号 + 大字
- 列表项用符号或图标标记

**验证**：全文可滚动浏览，各章节视觉层次分明

### Phase 4：滚动动画

- Intersection Observer 驱动入场动画
- 章节内容渐入（fade + 小幅上移）
- 代码卡片依次出现
- 避免过度动效，保持演示场景的沉稳感
- `prefers-reduced-motion` 适配

**验证**：滚动流畅，动画不拖沓，不影响投屏可读性

### Phase 5：投屏适配和收尾

- 投屏常用分辨率测试
- 导航栏在小屏上的处理
- 确保代码块不溢出
- 打印样式（可选）
- 移动端基本可读（可选）

**验证**：在 1280×720 和 1920×1080 分辨率下视觉效果合格

## 技术范围

- 单文件 HTML + 内联 CSS + 少量 vanilla JS
- 无框架、无构建步骤
- 不引入外部字体（使用系统字体栈）
- 可以直接在浏览器打开

## 风险和注意事项

- **内容长度**：15 章节全部放入单页会很长，需要通过视觉节奏（留白、色块分隔）防止阅读疲劳
- **苹果风格 vs 投屏可读性**：大字 + 留白可能导致滚动距离剧增，需要在气势和实用性之间平衡
- **代码块**：大纲中有大量 prompt 示例代码块，需要确保投屏时等宽字体可读
- **后续 Skill 调整**：视觉方向可能会根据后续 Skill 的决策调整，第一期不宜定死所有细节
